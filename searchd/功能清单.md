实现一个 Windows 本地文件搜索引擎 MVP：Rust 作为核心服务，基于 NTFS MFT 构建文件名内存索引，通过 USN Journal 做增量更新；使用 Length + Protobuf 的本地 IPC（Named Pipe）对外提供 Search 服务；Qt(C++) 作为前端，仅实现搜索输入、结果列表与文件打开功能。

Windows 本地文件搜索引擎 MVP（Rust + Qt）

Protobuf 定义（固定，不得随意扩展）
syntax = "proto3";

package search.ipc;

service SearchService {
  rpc Ping (PingReq) returns (PingResp);
  rpc BuildIndex (BuildIndexReq) returns (BuildIndexResp);
  rpc Search (SearchReq) returns (SearchResp);
}

message PingReq {}

message PingResp {
  string version = 1;
}

message BuildIndexReq {
  repeated string roots = 1;
}

message BuildIndexResp {
  bool success = 1;
  uint64 indexed_files = 2;
}

message SearchReq {
  string keyword = 1;
  uint32 limit = 2;
}

message SearchResp {
  repeated SearchResult results = 1;IPC
}


message SearchResult {
  string path = 1;
  string filename = 2;
}

6️⃣ Rust 功能实现要求（逐条）
6.1 MFT 初始索引（MVP）

扫描 NTFS MFT

仅索引：

文件名

完整路径

不索引目录权限、时间、owner

输出到内存索引。

6.2 USN Journal（简化）

只处理：

FILE_CREATE

FILE_DELETE

RENAME_NEW_NAME

允许：

程序重启后全量重建

不持久化 checkpoint

6.3 内存索引结构（禁止变更）
HashMap<String, Vec<FileId>>

struct FileId {
    path: String,
    filename: String,
}


key = lowercase filename

不要 Trie

不要 FST

不要倒排索引

6.4 文件名搜索（MVP）

不区分大小写

子串匹配（contains）

返回前 N 条

6.5 文件内容搜索（Lite）

使用 ripgrep CLI

同步调用

返回文件路径即可

不建立内容索引

7️⃣ IPC Server 行为规范（Rust）

启动即监听 Named Pipe

支持：

Ping

BuildIndex

Search

单线程串行处理即可

出错返回空结果，不 panic


双击结果 → ShellExecute 打开文件
